Use `steel-skill-creator` when a recurring browser workflow should become an installable skill.

```bash Terminal
npx skills add steel-dev/skills --skill steel-skill-creator
```

## Authoring Rules

- Keep `SKILL.md` concise and task-oriented.
- Put detail in one-level `references/` files.
- Include at least three evals using the `assertions` key.
- Do not embed API reference copies that can drift.
- Do not include session IDs, trace IDs, timestamps, credentials, or raw logs.
- Add clear handoffs to `steel-session-debugging` and `steel-reliability`.

## Basic Structure

```text
my-browser-task/
  SKILL.md
  README.md
  references/
    workflow.md
    examples.md
    troubleshooting.md
  evals/evals.json
```

Use Steel traces to identify stable steps, parameters, selectors, waits, and success signals.
