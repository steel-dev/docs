Hey everyone — this week was mostly about making Steel easier to use from agent tooling and tightening up reliability across the platform. We also expanded the docs and examples around newer agent integrations, onboarding, and paid session flows.

### 🔧 Improvements

*   Updated Steel Cloud onboarding with a more agent-native setup flow and clearer API key guidance.

*   Added shell completion to Steel CLI.

*   Added new integrations and examples for AI SDK and OpenAI Agents across the docs and cookbook.

*   Added documentation for Steel's x402 pay-per-use browser sessions endpoint, including protocol flow, pricing, and supported payment networks.

*   Steel CLI now supports Codex, OpenCode, and Pi coding agents, with a simplified `--agent` flow for installing and using Steel agent skills.

*   Expanded integration docs for coding agents and enabled more Sessions API and integrations pages for `llms.txt` and copy-as-markdown sharing.

*   Refreshed computer-use and agent examples across the docs and cookbook, including updates to the Next.js integration example.

*   Updated docs for API key authentication, install command consistency, and current supported regions.

*   Improved internal admin console navigation and added quick links and copy actions for organization, Clerk, and Stripe identifiers.

*   Infrastructure updates improved rollout safety, browser VM sizing, fleet-controller tuning, and support for additional proxy and machine configurations.

### 🐛 Bug Fixes

*   Fixed session metering so billed duration is clamped to the configured session timeout.

*   Fixed the `forge` command in Steel CLI.

*   Resolved several docs issues, including incorrect JSX, sidebar problems, and broken links.


As always, thanks for testing out the Steel beta. We really look forward to more of your feedback and continuing to build with this awesome, curious, and supportive community.
